# Документация проекта для Codex

## Назначение

Этот каталог содержит вспомогательную проектную документацию для Codex и других AI-агентов, которые будут разрабатывать веб-приложение для учета НК сварных соединений.

Документы не заменяют корневые файлы проекта. Перед началом работы агент должен читать их в таком порядке:

1. `README.md` — краткое описание проекта, стека, модулей и базовых правил.
2. `AGENTS.md` — обязательные правила работы AI-агентов.
3. `docs/project.md` — основной проектный документ.
4. Профильный документ из `docs/`, соответствующий задаче.

## Общие правила для всех документов

1. Не выдумывать сущности, поля, статусы, роли и бизнес-правила.
2. Не менять утвержденные термины без прямой команды.
3. Не усложнять архитектуру без прямой необходимости.
4. Не превращать систему в микросервисную архитектуру.
5. Не делать тяжелую SPA-архитектуру базовым решением.
6. Не подменять термин `Дешифровщик` словом `Расшифровщик`.
7. Не хранить методы контроля строкой через запятую.
8. Не хранить файлы в БД.
9. Не отдавать приватные файлы напрямую из публичной директории.
10. Не редактировать утвержденные заключения напрямую.
