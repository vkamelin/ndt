# Карта интерфейса

## 1. Назначение

Документ фиксирует основные экраны веб-приложения и PWA-сценарии. Codex должен использовать его при создании маршрутов, Livewire-компонентов, Blade-шаблонов и навигации.

## 2. Общие правила UI

1. Интерфейс должен быть рабочим, быстрым и плотным.
2. Не делать декоративную сложность.
3. Основные элементы: таблицы, фильтры, карточки, формы, модалки, журналы, статусы, действия.
4. Все большие списки — с серверной пагинацией.
5. Фильтры должны выполняться на стороне БД.
6. UI может скрывать недоступные действия, но backend обязан проверять права через policy.
7. Для Android/планшетов использовать PWA-экраны.

## 3. Основная навигация

Рекомендуемые разделы:

- Рабочий стол;
- Заявки;
- Стыки;
- Задания;
- Результаты контроля;
- Заключения;
- Смены;
- Оборудование;
- Сотрудники;
- Документы и архив;
- Справочники;
- Администрирование.

## 4. Рабочий стол

Показывает:

- активные заявки;
- просроченные задания;
- задания текущего пользователя;
- заявки на утверждении;
- незавершенные смены;
- предупреждения по оборудованию;
- предупреждения по квалификациям;
- последние уведомления.

Правила отображения:

- администратор видит данные по всем объектам/участкам;
- начальник участка видит данные своего объекта/участка;
- исполнитель видит свои задания и смены.

## 5. Заявки

### 5.1 Список заявок

Фильтры:

- город;
- объект/участок;
- статус;
- дата заявки;
- срок выполнения;
- заказчик;
- номер заявки;
- метод контроля.

Колонки:

- номер;
- дата;
- город;
- объект/участок;
- заказчик;
- статус;
- срок;
- количество стыков;
- действия.

### 5.2 Карточка заявки

Вкладки:

- общие данные;
- стыки;
- методы контроля;
- задания;
- результаты;
- заключения;
- файлы;
- история изменений.

## 6. Стыки

### 6.1 Список стыков

Фильтры:

- город;
- объект/участок;
- заявка;
- чертеж;
- линия;
- номер стыка;
- статус;
- метод контроля;
- результат;
- заключение;
- баркод.

### 6.2 Карточка стыка

Вкладки:

- общие данные;
- методы контроля;
- задания;
- результаты;
- дефекты;
- ремонт / повторный контроль;
- заключения;
- архив;
- история.

## 7. Задания

Разделы:

- мои задания;
- задания по объекту/участку;
- задания по исполнителю;
- задания по методу контроля;
- просроченные задания.

Основные действия:

- создать задание;
- назначить исполнителя;
- принять в работу;
- отметить выполнение;
- вернуть;
- отменить.

## 8. Результаты контроля

Разделы:

- РК;
- ВИК;
- ПВК;
- МК;
- УК;
- результаты на анализе;
- результаты с дефектами;
- результаты, требующие пересвета;
- результаты, готовые к заключению.

Карточка результата должна показывать:

- стык;
- метод;
- исполнитель;
- дата;
- оборудование;
- НТД;
- результат;
- дефекты;
- файлы;
- связь с заключением;
- история изменений.

## 9. Радиографический контроль

Для РК нужны отдельные экраны или вкладки:

- пленки;
- снимки;
- пересветы;
- плотности;
- дешифровка;
- архив;
- реестры передачи.

Статусы РК должны быть видны пользователю.

## 10. Заключения

Раздел заключений:

- список;
- карточка;
- создание;
- проверка;
- утверждение;
- выдача;
- версии;
- печать PDF;
- связанные стыки и результаты.

После утверждения действия редактирования должны быть недоступны. Исправления — только через версию, замену, аннулирование или корректирующую запись.

## 11. Смены

Разделы:

- текущая смена;
- смена лаборанта;
- смена дешифровщика;
- журналы смен;
- отчеты смен;
- мои смены;
- смены по объекту/участку;
- смены по сотруднику.

### 11.1 Экран смены лаборанта

Действия:

- старт смены;
- прием пленки;
- выдача пленки;
- списание пленки;
- запрос химии;
- поступление химии;
- замена химии;
- регламентные работы;
- передача материалов на дешифровку;
- завершение смены.

### 11.2 Экран смены дешифровщика

Действия:

- старт смены;
- очередь пленок/снимков;
- фиксация просмотренных групп;
- фиксация брака;
- фиксация подозрения на подлог;
- очистка рабочего места;
- результат дешифровки;
- передача результата на анализ;
- завершение смены.

- архив;
- поиск.

## 15. PWA-экраны

### 15.1 Дефектоскопист: “Мои задания НК”

Функции:

- список назначенных заданий;
- фильтр по объекту/участку и дате;
- карточка стыка;
- отметка доступности стыка;
- отметка выполнения контроля;
- ввод результата;
- загрузка фото/файлов;
- сканирование QR/штрихкода;
- комментарий;
- передача результата дальше.

### 15.2 Лаборант: “Смена лаборанта”

Функции:

- старт смены;
- прием пленки;
- выдача пленки;
- списание пленки;
- запрос химии;
- поступление химии;
- замена химии;
- регламентные работы;
- сканирование баркода;
- завершение смены;
- сменный отчет.

### 15.3 Инженер НК / Дешифровщик: “Дешифровка”

Функции:

- очередь пленок/снимков;
- карточка материала;
- отметка просмотра;
- результат дешифровки;
- дефекты;
- брак;
- подозрение на подлог;
- очистка рабочего места;
- передача результата на анализ;
- завершение смены дешифровщика.

### 15.4 Начальник участка: “Контроль участка”

Функции:

- активные заявки по объекту/участку;
- просроченные задания;
- быстрые статусы;
- утверждение или возврат результата;
- утверждение заключения;
- уведомления;
- критичные отклонения.
